# 既知の罠カタログ（レビュアー監査用）

subject（イベントソーシング在庫管理）に意図的に仕込まれた、または coder が踏みやすい既知の欠陥。
**このファイルは combo ディレクトリにコピーされない**（レビュアーに見えるとカンニングになる）。
監査では「最終コードに残っているか」と「レビューレポートで指摘されたか」を突き合わせる。

| ID | 罠 | 説明 | テストでの検出 |
|----|-----|------|---------------|
| T1 | ShipStock の引当追跡 | StockShipped イベントは数量を持たないため、projection は引当数量を自前で追跡する必要がある。「全引当合計を減算」「全引当をクリア」は複数引当中の部分出荷で在庫を破壊する | **不可**（複数引当中の部分出荷テストは存在しない） |
| T2 | available の二重管理 | available は onHand − 引当合計の導出値。状態として保持すると更新漏れでズレる | 部分的 |
| T3 | version の二重管理 | イベントストアの version はイベント配列長から導出可能。別カウンタを持つと不整合の余地 | 不可 |
| T4 | 依存方向違反 | ドメイン（decide/evolve）がストアやプロジェクションに依存する、CommandHandler が具象ストアに依存する等 | 不可 |
| T5 | モノリス | 全実装を index.ts に詰め込み、モジュール分割しない（README は分割を設計判断として委ねるが、複数責務の単一ファイルは arch 知識の REJECT 対象） | 不可 |
| T6 | initialState の重複/可変 | initialState を複数箇所で再定義する、mutable なまま公開する | 不可 |
| T7 | evolve の不純 | evolve が throw する、引数 state を直接変更する、未知イベントを黙って握りつぶす設計判断の説明がない | 部分的（mutate はテストあり） |
| T8 | テスト/契約ファイルへの変更要求 | tests/ と src/types.ts は変更禁止。これらへの修正を要求する指摘はスコープ外であり、fix 不能な指摘としてループを空転させる | - |
