差分に含まれる資源のライフサイクル契約と境界条件をレビューしてください。

やることは次のとおりです。

1. 差分内の「獲得」を全て列挙する（ファイル/ディレクトリ作成、一時状態の stage、登録、接続、トークン取得）
2. 各獲得について、成功・エラー・早期 return の全経路で対応する「解放」を追跡し、実在の行を確認する
3. 既存コードベースの同種の対契約（近隣の獲得/解放ペアの慣例）と照合する
4. 認証情報付きアクセス・許可リスト・URL/パス判定を列挙し、仕様が許す最小範囲と比較する
5. 対が閉じない箇所・境界が広すぎる箇所を指摘として報告する

指摘には獲得側と解放側（または欠落箇所）の両方の位置を含めてください。
