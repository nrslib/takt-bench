# cron-next

5 フィールドの cron 式を解析し、次回実行時刻を計算するライブラリ。

## 仕様

`src/cron.ts` に次の 2 つの関数を実装する。

### `parseCron(expr: string): CronSchedule`

cron 式（`分 時 日 月 曜日` の 5 フィールド、空白区切り）を解析して `CronSchedule` を返す。

対応する記法（全フィールド共通）:

| 記法 | 例 | 意味 |
|------|-----|------|
| `*` | `*` | すべての値 |
| 数値 | `5` | その値のみ |
| リスト | `1,15,30` | 列挙した値 |
| 範囲 | `9-17` | 9 から 17 まで |
| ステップ | `*/15`, `10-30/7` | 範囲内を n 刻み |

フィールドの値域:

| フィールド | 値域 |
|-----------|------|
| minutes | 0-59 |
| hours | 0-23 |
| daysOfMonth | 1-31 |
| months | 1-12 |
| daysOfWeek | 0-7（0 と 7 は日曜。7 は 0 に正規化する） |

返り値の各配列は昇順ソート・重複排除済みであること。`domRestricted` / `dowRestricted` は、日フィールド / 曜日フィールドが `*`（またはステップ付き `*`。例 `*/2`）以外で制限されているとき true。

不正な式（フィールド数が 5 以外、値域外、逆転した範囲 `5-1`、ステップ 0、数値でないトークンなど）は `Error` を throw する。

### `nextRun(expr: string, from: Date): Date`

`from` より厳密に後（strictly after）で、式にマッチする最初の時刻を UTC 基準で返す。

- 候補は分単位（秒・ミリ秒は 0）
- `from` 自身がマッチしても返さない（strictly after）
- 日と曜日が両方制限されている場合は OR 条件（どちらかにマッチする日）— 標準 cron 準拠
- 片方だけ制限されている場合は制限されている方のみで判定
- 引数の `from` を変更（mutate）しないこと
